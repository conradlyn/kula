# Internationalization (i18n)

Package: [`internal/i18n`](../../internal/i18n/) — file
[`i18n.go`](../../internal/i18n/i18n.go).

Kula ships translations for the web dashboard. Locale JSON files are embedded into the binary
with `//go:embed locales/*.json`, so there's nothing to install or ship separately.

## Locales

There are **26** embedded locale files under
[`internal/i18n/locales/`](../../internal/i18n/locales/):

```
ar bn cs de en es fr he hi id it ja ko ms nl pl pt ro ru sv th tr uk ur vi zh
```

The full set is declared in `SupportedLangs`. Each file is a flat JSON map of string keys to
translated strings, for example (`en.json`):

```json
{
  "sign_in": "Sign In",
  "username": "Username",
  "cpu": "CPU",
  "load_avg": "Load Avg (1m)",
  ...
}
```

> Note: `web.lang.default` in the config example documents a smaller curated subset
> (`ar de en es fr hi ja ko pl pt zh`) for the UI selector; the backend `SupportedLangs` list is
> broader. Keep the `SupportedLangs` slice in `i18n.go` in sync with the dashboard's
> `i18n.js` `SupportedLangs`.

## Translator

```go
t := i18n.NewTranslator(lang)   // "" → auto-detect via DetectLang()
s := t.Get("sign_in")           // returns the translation, English fallback
```

- `NewTranslator(lang)` falls back to `"en"` if `lang` isn't supported, and to `DetectLang()`
  (environment-based) when `lang` is empty.
- `load()` **always** loads `en.json` first as a fallback map, then overlays the requested
  language. Any key missing in the target language resolves to the English string — so a partial
  translation never shows a blank or raw key.

## Serving translations to the UI

The dashboard fetches the active language's strings from `GET /api/i18n?lang=<code>`
([handleI18n](../../internal/web/server.go)), which rejects junk/traversal language codes. The
frontend [`i18n.js`](../../internal/web/static/js/app/i18n.js) applies the returned map to the
DOM.

The default language and whether the selector is shown are controlled by `web.lang.default` and
`web.lang.force` (see [Configuration](../user/04-configuration.md#web-display-options)).

## Adding or updating a translation

1. Edit the relevant `locales/<code>.json` (add any new keys present in `en.json`).
2. To add a **new language**: create `locales/<code>.json`, add `<code>` to `SupportedLangs` in
   `i18n.go`, and add it to the dashboard's `i18n.js` `SupportedLangs` (and the config docs if it
   should appear in the curated UI list).
3. Run `./addons/check.sh` — `i18n_test.go` validates the locale set.

Because the files are embedded, a rebuild is required for changes to take effect.
